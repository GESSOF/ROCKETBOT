🌐 Comandos Web - Automatización de Navegadores

Conjunto completo de comandos para automatizar navegadores web, interactuar con elementos DOM, manejar formularios y realizar scraping de datos.

🌐
Open Browser - Abrir Navegador
▼
Función: Inicia un navegador web y navega a una URL específica. Es el comando fundamental para iniciar cualquier automatización web.

📋 Parámetros Principales:

  • Browser: Chrome, Firefox, Edge, Safari
  • URL: Dirección web de destino
  • ID Session: Identificador para múltiples navegadores
  • Profile: Perfil del navegador (para mantener sesiones)

💡 Ejemplo de Uso:

Browser: Chrome
URL: https://www.google.com
ID Session: main_browser
Result: resultado_apertura
⚠️ Importante: Siempre validar la apertura del navegador con "Wait for Object" después de este comando.
⏳
Wait for Object - Esperar por Objeto
▼
Función: Espera a que un elemento web específico aparezca, sea visible o clickeable. Comando crucial para validaciones.

📋 Parámetros:

  • Object: XPath, ID, Class del elemento
  • Type: xpath, id, class, name
  • Action: visible, clickable, present
  • Timeout: Tiempo máximo de espera (segundos)
✅ Uso Obligatorio: Usar después de Open Browser y antes de cualquier interacción con elementos web.
👆
Click Object - Hacer Clic en Objeto
▼
Función: Realiza clic en un elemento web específico como botones, enlaces o cualquier elemento clickeable.

📋 Parámetros:

  • Object: Selector del elemento
  • Type: xpath, id, class, name
  • Click Type: left, right, double

✨ Diferencia XPath vs ID:

  • XPath: Ubicación exacta del elemento
  • ID: Identificador único (preferible si es fijo)
  • Full XPath: Ruta completa (menos confiable)
  • Class: Para múltiples elementos similares
⌨️
Send Text - Enviar Texto
▼
Función: Escribe texto en campos de entrada, formularios o cualquier elemento editable de la página web.

💡 Ejemplo Búsqueda en Google:

Object: //input[@name='q']
Text: {variable_busqueda}
Clear First: True
Send Enter: True
📄
Get Text - Extraer Texto
▼
Función: Extrae el texto contenido en un elemento web específico para procesarlo o guardarlo.

✨ Uso Común:

  • Extraer precios de productos
  • Obtener títulos de páginas
  • Capturar mensajes de confirmación
  • Leer datos de tablas
💻
Execute JavaScript - Ejecutar JavaScript
▼
Función: Ejecuta código JavaScript personalizado en la página web actual para acciones avanzadas.

💡 Ejemplos de JavaScript:

// Extraer por clase
return document.getElementsByClassName("precio")[0].innerText;

// Abrir nueva ventana
window.open('https://example.com', '_blank');

// Scroll hasta elemento
document.getElementById("footer").scrollIntoView();

🧠 Comandos Lógicos - Control de Flujo

Estructuras de control para toma de decisiones, bucles, manejo de errores y control del flujo de ejecución del robot.

❓
IF - Condición Si/Entonces
▼
Función: Ejecuta diferentes bloques de código según se cumpla o no una condición específica.

📋 Operadores Disponibles:

  • ==Igual a
  • !=Diferente de
  • >Mayor que
  • <Menor que
  • ANDY lógico
  • ORO lógico
🔄
FOR - Bucle de Iteración
▼
Función: Repite un bloque de código para cada elemento de una lista o rango específico.

💡 Estructura de Lista de Excel:

datos = [
["Juan", "juan@email.com", 123456],
["Ana", "ana@email.com", 789012],
["Carlos", "carlos@email.com", 345678]
]

# En FOR loop:
Iterable: datos
Item Variable: fila

# Dentro del loop:
nombre = fila[0]
email = fila[1]
telefono = fila[2]
🛡️
TRY CATCH - Manejo de Errores
▼
Función: Ejecuta un bloque de código y maneja errores si ocurren. Diferente del IF.

💡 Diferencia con IF:

# IF: Ejecuta UNO u OTRO
if condicion:
# Bloque A
else:
# Bloque B

# TRY CATCH: Ejecuta TRY, CATCH solo si error
try:
# Siempre se ejecuta
catch:
# Solo si hay error en TRY

📊 Comandos Excel - Automatización de Hojas de Cálculo

Conjunto completo de comandos para trabajar con archivos Excel, leer y escribir datos, aplicar formatos y ejecutar macros.

📊
Open Excel vs Open XLSX
▼

Excel: Requiere Microsoft Excel instalado. Abre la aplicación visualmente.

XLSX: No requiere Excel instalado. Funciona en Mac, Linux y Windows sin licencia.

✨ Recomendación:

  • Preferir XLSX para mayor compatibilidad
  • Excel solo si necesitas macros específicas
  • XLSX no abre aplicación en segundo plano
📖
Read Cells - Leer Celdas
▼
Función: Lee un rango de celdas de una hoja de Excel y guarda los datos en una variable.

💡 Rango Dinámico vs Fijo:

# ❌ Rango fijo (malo)
Range: A2:D6

# ✅ Rango dinámico (bueno)
Range: A2:D{filas}

# Primero contar filas
Count Rows → variable: filas
# Luego leer con rango dinámico

🗄️ Comandos MySQL - Gestión de Bases de Datos

Comandos para conectar, consultar y manipular bases de datos MySQL, ejecutar queries y gestionar transacciones.

🔌
Connect MySQL - Conectar Base de Datos
▼
Función: Establece conexión con servidor MySQL para ejecutar consultas y operaciones.

📋 Parámetros de Conexión:

  • Host: Servidor de base de datos
  • Port: Puerto (generalmente 3306)
  • Database: Nombre de la base de datos
  • User: Usuario de acceso
  • Password: Contraseña del usuario
🔍
Execute Query - Ejecutar Consulta
▼
Función: Ejecuta consultas SQL (SELECT, INSERT, UPDATE, DELETE) en la base de datos conectada.

💡 Ejemplos de Consultas:

# SELECT - Consultar datos
SELECT * FROM usuarios WHERE activo = 1

# INSERT - Insertar registro
INSERT INTO productos (nombre, precio) VALUES ('Laptop', 1500)

# UPDATE - Actualizar datos
UPDATE usuarios SET ultimo_acceso = NOW() WHERE id = 123

# DELETE - Eliminar registro
DELETE FROM pedidos WHERE fecha < '2023-01-01'
🔒
Close MySQL - Cerrar Conexión
▼
Función: Cierra la conexión con la base de datos para liberar recursos.
✅ Buena Práctica: Siempre cerrar conexiones DB al final del robot para evitar conexiones huérfanas.

🌍 Comandos HTTP - Conexiones Web y APIs

Herramientas para realizar peticiones HTTP, consumir APIs REST, enviar datos POST/GET y manejar respuestas JSON.

📥
HTTP Request - Petición HTTP
▼
Función: Realiza peticiones HTTP a APIs y servicios web con diferentes métodos (GET, POST, PUT, DELETE).

📋 Parámetros Principales:

  • URL: Endpoint de la API
  • Method: GET, POST, PUT, DELETE, PATCH
  • Headers: Cabeceras HTTP (Content-Type, Authorization)
  • Body: Datos a enviar (JSON, XML, form-data)
  • Timeout: Tiempo máximo de espera

💡 Ejemplos de Peticiones:

# GET - Obtener datos
URL: https://api.ejemplo.com/usuarios
Method: GET
Headers: {'Authorization': 'Bearer token123'}

# POST - Enviar datos
URL: https://api.ejemplo.com/usuarios
Method: POST
Headers: {'Content-Type': 'application/json'}
Body: {"nombre": "Juan", "email": "juan@email.com"}
📊
Parse JSON - Procesar JSON
▼
Función: Convierte respuestas JSON en estructuras de datos manejables para extraer información específica.

💡 Ejemplo de Procesamiento:

# Respuesta JSON de API
{
"usuarios": [
{"id": 1, "nombre": "Juan", "activo": true},
{"id": 2, "nombre": "Ana", "activo": false}
]
}

# Extraer primer usuario
primer_usuario = json_data["usuarios"][0]
nombre = primer_usuario["nombre"] # "Juan"
📁
Download File - Descargar Archivo
▼
Función: Descarga archivos desde URLs remotas y los guarda en el sistema local.

✨ Casos de Uso:

  • Descargar reportes de APIs
  • Obtener archivos de respaldo
  • Sincronizar documentos remotos
  • Descargar imágenes y media

▼
